Feedback and Program Direction

Technical Description

I created the Feedback and Program Directory page using the shared GreenSteps header, navigation bar and footer. The main page content was divided into three clear sections: a horizontally scrollable programme directory, a ratings and reviews section, and an interactive feedback form.

The programme directory was created using semantic section elements and individual programme cards. Each card contains a relevant image with alternative text, a programme title, a short description and an action button. CSS Flexbox was used to display the cards in one horizontal row, while overflow-x: auto allowed users to scroll through the available programmes. Hover effects, shadows, rounded corners and the GreenSteps colour palette were used to make the cards visually interactive and consistent with the shared website design.

The reviews section contains an overall numerical rating and three individual review cards. Each review includes a title, reviewer details, date, star rating, short comment and a See More link. Flexbox and responsive wrapping were used to organise the review cards across different screen sizes.

The feedback form uses fieldset and legend elements to organise personal information and programme feedback. It includes text, email, telephone, date, datetime-local, select and textarea fields. JavaScript event listeners were used to create a live character counter, check missing or invalid values, display error messages beside the relevant fields and show a successful submission confirmation. The form does not send information to a server.

I referred to the MDN Web Docs guide on client-side form validation when implementing required-field checking, custom error messages and JavaScript validation [1].

Design and Implementation Rationale

I selected Flexbox for the programme and review layouts because it supports flexible alignment, horizontal scrolling and responsive wrapping. Fieldset and legend were used because they visually and semantically group related form controls. The green colour palette, rounded cards and subtle shadows maintain consistency with the environmental theme of the GreenSteps website.

Accessibility

Every programme and reviewer image includes meaningful alternative text. All form controls have visible labels, and related fields are grouped using fieldset and legend. Error messages appear close to the relevant fields, keyboard focus styles remain visible and the page uses readable colour contrast. The layout also responds to smaller screens.

Team Page

Technical Description

I created the Team Page to introduce the four members of the GreenSteps group and explain each person's role and individual contribution. The page uses one interactive member block for each student. Every block contains a thumbnail image, the student's name, their group role and an expandable details area.

CSS Grid was used to organise the four member blocks into a balanced responsive layout. Hover effects were applied using transformations, border changes and shadows to show that each member block is interactive. The additional contribution details are revealed using CSS properties such as opacity, transform, visibility and transition.

JavaScript was not required for this page because the coursework requested CSS-based hover and focus interaction. The same expanded details can be displayed when a keyboard user focuses on a member block.

Design and Implementation Rationale

A card-based layout was selected because it gives equal visual importance to every group member. CSS Grid makes the layout easy to rearrange for tablets and mobile screens. Smooth transitions were used to reveal the details without creating sudden changes in the page layout.

Accessibility

Each member image includes meaningful alternative text. The interactive containers include tabindex="0", allowing keyboard users to reach them using the Tab key. The focus state is clearly visible, and the same details shown on hover are also available during keyboard focus. Names and roles are provided as text rather than being placed only inside images.

Climate Change and Young People Content Page

Technical Description

My individual Content Page focuses on Climate Change and Young People. It explains how students and younger generations can support climate action through sustainable lifestyles, greener transport, community participation, education and technology.

The page uses one main h1 heading followed by logically ordered h2 and h3 headings. The content is divided into separate section elements, and a page navigation menu uses internal anchor links to move users directly to each topic. Every destination section has a unique id that matches its navigation link.

CSS Grid was used to place text and images side by side on desktop screens. Alternate sections reverse the image and text positions to create visual variety. Media queries change these layouts into a single column on tablets and mobile devices.Images were styled with responsive widths, fixed display heights, object-fit, borders, rounded corners and shadows.

The page-specific CSS was embedded inside the HTML document, while the shared style.css file continues to control the common header, navigation, footer and Top button. Smooth scrolling was added for internal navigation. No JavaScript was required for the Content Page.

Design and Implementation Rationale

Internal navigation was included because the page contains several long sections. This helps users reach specific content without scrolling through the entire page. Figure and figcaption elements connect each image to a short explanation, while responsive Grid layouts make the page readable across different devices.

Accessibility

The page uses a logical heading hierarchy, meaningful section IDs, descriptive link text and alternative text for every image. Figure captions provide additional context. Keyboard focus indicators are visible on internal links, and the fixed Top button allows users to return to the beginning of the page.Text and images stack into a single column on smaller screens.

Challenges and Lessons Learned

One challenge was creating a programme directory that remained horizontally scrollable without causing the entire Feedback Page to overflow. I resolved this by applying overflow-x only to the programme container and using fixed flexible card widths.I also adjusted the card heights, image dimensions and spacing to create a consistent layout.

Another challenge was implementing custom form validation while keeping the error messages close to their related fields. I created separate error elements for the form controls and used JavaScript to update them when the user submitted missing or invalid information. This helped me understand event listeners, input validation and DOM manipulation more clearly.

On the Team Page, I needed to ensure that information available through mouse hover was also available to keyboard users.Adding tabindex and focus-based CSS helped me create a more inclusive interaction.

Creating the Content Page improved my understanding of CSS Grid, responsive media queries, internal anchor navigation and semantic content organisation. I also learned the importance of keeping shared styles in style.css while limiting page-specific styles to the appropriate page.

Accessibility Considerations

Accessibility was built into all three pages from the start. Each page uses semantic landmarks, header, nav, main and footer, with a logical heading order and aria-labels, so screen readers can map the page and users can jump between sections. Every meaningful image has descriptive alt text, decorative layers are hidden with aria-hidden, and the content page pairs images with figure and figcaption.

Colour is handled carefully: the shared green palette was chosen for good contrast, the hero uses a dark overlay so white text stays readable over the photo, and colour is never the only cue, since focus outlines, movement and shape reinforce it for colour-blind users. Every interactive element works by keyboard, with a skip link on each page, visible focus-visible outlines, and hover states matched by focus states. The JavaScript-generated gallery cards were given a tabindex and Enter and Space key support so they behave like real buttons.

Compliance

The pages I developed follow responsible web publishing practices. The written content is suitable for an educational audience and focuses on constructive climate action. I avoided inappropriate, misleading or discriminatory material.

External images and resources used on the pages will be recorded in the References section with their original sources. I did not claim ownership of external material. The written content was produced specifically for this coursework rather than copied directly from the United Nations or other websites.

The Feedback form is used only to demonstrate client-side interaction and does not transmit or permanently store personal information. Navigation, accessibility and validation were tested to support responsible and professional publication.
